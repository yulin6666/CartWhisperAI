## 1. 产品概述

### 1.1 核心价值主张

CartWhisper AI 是一个智能推荐引擎，使用 DeepSeek AI 分析商品属性，自动生成高度相关的跨品类推荐。

**关键优势**：
- 🤖 AI 驱动的推荐引擎（无需手动配置）
- 🔗 跨品类推荐（鞋子推荐包包，不会重复推荐鞋子）
- 📊 完整的推荐管理仪表盘
- 🎯 提升平均订单值（AOV）和转化率

### 1.2 目标客户

| 客户群体 | 特征 | 优先级 |
|--------|------|------|
| **初创卖家** | 1-50 个商品, 月销 < $10K | 高 (免费版) |
| **中小商家** | 50-500 个商品, 月销 $10K-100K | 中 (Pro 版) |
| **成长型商家** | 500+ 个商品, 月销 > $100K | 低 (Max 版, 二期) |

## 3. 免费版 vs Pro 版 详细对比

### 3.1 为什么免费版限制 50 个商品?

**市场数据分析**:
根据 2025 年 Shopify 统计：
- 27.6% 店铺：1-9 个商品
- 17.4% 店铺：10-24 个商品
- 12.9% 店铺：25-49 个商品
- **累计 57.9% 店铺 < 50 个商品**
- 剩余 42.1% 店铺 50-10,000+ 个商品

**为什么选择 50 个？**:
- 50 个商品 = 覆盖约 58% 的初创店铺
- 足够体验：让用户完整体验 AI 推荐功能
- 升级压力适中：42% 店铺超过 50 个商品，形成自然升级动力
- 避免过度慷慨：100 个太多（覆盖 75%），导致大多数用户永远不需要升级
- 升级点清晰：一旦超过 50 个，立即提示升级

**为什么推荐数改成 1 vs 2？**:
- 1 个推荐 = 足以让用户看到价值和质量
- 缺少第 2 个 = 明显的升级激励（不是完全限制）
- 两个都不多不少 = 重点放在"编辑能力"而非"数量"

### 3.2 升级激励点（Pro 版的真实价值）

| 痛点 | 免费版体验 | Pro 版解决 | 升级概率 |

### 3.4 推荐刷新策略 (重要 - API 成本优化)

**问题分析：为什么不能每日自动刷新？**
```
DeepSeek API 成本估算:
├─ 每个商品 1 次 API 调用 (生成推荐)
├─ 50 个商品 ≈ 25,000-50,000 tokens ≈ $0.01-0.02
├─ 1000 个商品 ≈ $0.1-0.3
│
├─ 每日刷新成本 (假设 1000 用户):
│   ├─ 每天: 1000 × $0.05 = $50
│   └─ 每月: $1,500 😱
│
└─ 结论: 每日自动刷新不可行
```

**新策略：增量更新 + 手动刷新**
```
刷新时机:
├─ 首次同步: 为所有商品生成推荐 (必须)
├─ 增量同步: 只为新增商品生成推荐 (默认行为)
├─ 商品删除: 自动删除相关推荐
├─ 手动刷新: 用户点击"重新生成全部"
│   ├─ 免费版: 每月 1 次
│   └─ Pro 版: 每周 1 次
└─ 取消自动每日刷新 ❌
```

**优化后成本**:
```
├─ 首次同步 50 商品: $0.02
├─ 每月新增 5 商品: $0.005
├─ 每商店每月: ~$0.03
├─ 1000 用户每月: ~$30 (原方案 $1,500)
└─ 成本降低: 50 倍
```

### 3.5 API 限额与前端缓存

**使用场景分析**:
```
店铺流量与 API 调用 (无缓存):
├─ 小型店铺: 300 UV/天 × 3 页面 = 900 次
├─ 中型店铺: 1000 UV/天 × 3 页面 = 3000 次
└─ 大型店铺: 5000 UV/天 × 3 页面 = 15000 次
```

**解决方案：API 限额 + 前端缓存**
```
API 限额 (已调整):
├─ 免费版: 5,000 次/天 (原 1,000)
└─ Pro 版: 50,000 次/天 (原 10,000)

前端缓存 (必须实现):
├─ 浏览器 localStorage 缓存推荐结果
├─ 缓存时长: 1 小时
├─ 同一用户同一商品不重复请求
└─ 实际 API 调用减少 70-80%

组合效果:
├─ 5,000 限额 + 缓存 ≈ 支持 5,000-15,000 UV/天
└─ 50,000 限额 + 缓存 ≈ 支持 50,000-150,000 UV/天
```

```javascript
// Shopify Billing API 伪代码框架

// 第一步：检查用户订阅状态
async function checkSubscription(shop) {
  const query = `
    query {
      appInstallation {
        activeSubscriptions {
          name
          lineItems { id }
        }
      }
    }
  `;
  return await shopify.query(query);
}

// 第二步：根据订阅版本限制功能
function getFeatureAccess(subscription) {
  const isProPlan = subscription?.some(s => s.name === 'Pro Plan');

  return {
    maxProducts: isProPlan ? Infinity : 50,
    aiRecommendations: true, // 都有 AI
    recommendationsPerProduct: isProPlan ? 2 : 1,
    editableReasons: isProPlan, // Pro 可编辑
    apiLimit: isProPlan ? 50000 : 5000,
    manualRefreshLimit: isProPlan ? 4 : 1, // 每月刷新次数
  };
}

// 第三步：监听订阅状态变化
app.webhook('APP_SUBSCRIPTIONS_UPDATE', async (req, res) => {
  const { body } = req;
  const plan = body.appSubscription.name;

  await updateUserPlan(body.shop_id, plan);
  res.status(200).send();
});
```

```
DeepSeek API 成本 (增量更新，不再每日刷新):
├─ 首次同步 50 商品: ~$0.02 (一次性)
├─ 每月新增 5 商品: ~$0.005
├─ 手动刷新 (免费版每月1次): ~$0.02
└─ 每用户每月: ~$0.03

免费用户成本:
  - 1000 个免费用户 × $0.03 = $30/月

付费用户成本:
  - 50 个付费用户 × $0.05 = $2.5/月 (Pro 每周1次刷新)

总变动成本: ~$32.5/月
```

**收入 (目标)**:
```
Year 1: 50 个付费用户 × $19.99 = $1,000/月 = $12,000/年
年利润 = $12,000 - ($21×12) - ($32.5×12) = $12,000 - $642 = $11,358 ✅
```

**盈亏平衡分析** (改进后):
```
需要付费用户数: (固定成本 + 变动成本) / 收入
              = ($21 + $32.5) / $19.99
              ≈ 2.7 个付费用户/月

结论: 需要至少 3 个稳定付费用户即可达到盈亏平衡！
     (API 成本极低，主要是固定成本)
```
